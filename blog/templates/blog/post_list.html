Understanding an html Page

Starting the html file

  1. First thing to do is to save the file with a filename that literally says .html which sets it up to be read by the browser as an html file.
  2. After saving it as .html, you can type your first line, which will just be "html" then tab. This sets up the html element. this is the structure for the entire code you write for the page.

In the HEAD you write all that will not be seen by the user.

  • Note that anything typed will actually be visualized if you dont give the correct tags. So, for instance, the head is where you put the title tag
  • In the TITLE, you write what will tag the page in the search engine.

In the BODY you write your text. Each tag you give the text you write in the body will set up how it gets positioned on the page.

Now I will list the types of tags you can use in the body of your html file.

  1. to make the text bold, make a strong tag
  2. to make the text italicized, make an em tag
  3. to make a new line within a block of text, use a p tag
  4. the heading tags will look as follows:

    h1 tags

    h2 tags

    h3 tags

    h6 tags
  5. to make a numbered list, use an (ordered list) li tag
  6. to make a bulleted list, use an (un-orderd list) ul tag
  7. to post an image, you use a self-closing tag called "img". This means that all the code you input for the image is between the "<" and ">".
    • type img and tab
    • or alterernatively type "<" then "img" type a space and the word src= next to img. this is the source tag
    • next to the word "src=" paste the link for the image in quotation marks and then close the tag.

    • To change the size of the image you must add a qualifier to the tag before you close it. So you type [alt="Image" height="200" width="200"] and play with the numbers to get the right size.

    • also you have the option of encoding an image saved on your computer. Save the image in the same location that the code editor page is saved and then link it instead of an online link.

  8. to insert a link, you use an anchor tag which is just "a" then tab. this is not a self closing tag! The reason it needs a closing tag is that you are typing text in between the open and close portions, this is text that will be visible, it is what the link is "anchored" to, aka, the text that the user will click on to activate the link you have anchored.
    • the open anchor will say: a ref="http://www.something.com"
    • the space between the open and close anchor will say something like: "Click Here"
    • then there will be the closing tag
    • Click Here for MDN
  9. To make a table, you use a "table" tag which you have the option of attributing with a border by qualifying the table with an attribute before closing the tag. So you can say: table border="1"
    • You then define the table with a head element and body element, each of which will will set up with a thead or tbody tag and then the content is typed with a table row tag (tr) but when the content is typed, it differs in that the head must have each cell tagged with a (th) tag while the body has them tagged with a (td) tag. i believe this stands for table heading and table details but each item listed/tagged will appear in a cell to the right of the previous, so it lays out like rows rather than columns.
    • table border=1
    • table head (thead)
      • table row (tr)
      • table details for heading (th)
    • table body (tbody)
      • table row (tr)
      • table details for body (td)

Form tags

  1. a form tag has two key elements: 1) An Action = Where the form sends data to. 2) A Method = Will the data be sent to "GET" From or "POST" To.
  2. so you type a form tag and insert the action and method before closing the opening tag. If no action is specified, the submitted form will just refresh and if no method is specified, it will default to "GET". But a specified action can link to a URL and a specified method can Post to said URL
  3. then after closing the opening tag you can type in your input tags and then type the form closing tag
  4. input tags are already coded for their action in html but if you add a placeholder function within the action tag, it will allow you to display text on the server side
  5. for the action=text tag, if you add a placeholder it will put the words you type in your placeholder inside the button of the text box. When the user starts typing, that text disappears
    • input=text
    • input=password
    • input=color
    • input=button
    • input=submit

Labels

  1. Labels are helpful for instances like when someone is reading impaired. The label is associated with the form within the software and allows for it to be called out as such. This is also visually appealing.
  2. The label has two syntax forms. Using this example:

    • Directly entering the label just before entering the input tag and closing out the label tag after entering the input tag.

    • entering the label tag separately but connecting it with"for" and "ID" designations
